# QET 電氣圖繪製規範（AI Agent 執行版）

> **本文件的讀者是控制 QElectroTech（QET）MCP 工具的 AI agent。**
> 你的任務：依本規範產生／修改 `.qet` 專案、`.elmt` 元件與 `.titleblock` 圖框，使產出圖面符合 IEC 標準體系。
> 規則分級：**MUST**（違反即不合格，禁止輸出）、**SHOULD**（預設遵守，偏離須在回覆中說明理由）、**MAY**（允許的選項）。
> 適用標準：IEC 60204-1（機械電氣設備）、IEC 60617（圖形符號）、IEC 81346（參考標示）、IEC 61082-1（文件編製）、IEC 61355（文件分類）、ISO 7200 / ISO 5457（標題欄／圖紙）、ISO 13850（急停）。
> 本規範為公司內規整編版；與客戶合約指定標準衝突時，以合約為準，並回報衝突點。

## 1. 元件代號（IEC 81346）

### 1.1 格式

```
[=功能面][+位置面]-<類別字母><序號>[:端子號]
例：=G1+A1-K1:A1 → 功能群組G1、A1盤、繼電器K1 的 A1 端子
```

- 產品面前綴 `-` **MUST** 存在；`=` 與 `+` 面在單盤單機專案 **MAY** 省略。
- 序號自 1 起連續編號，同類別不得跳號、不得重用已刪除元件的代號（MUST）。
- 端子號使用元件實際印刷端子名（A1/A2、13/14、95/96…），不得自創（MUST）。

### 1.3 中文名稱標註（SHOULD）

- 元件除代號外，**應**適當加註中文功能名稱，格式：代號在上、中文名在下（或代號後空一格接中文），例：

```
-KM1          -S0           -Q1
主接觸器       急停按鈕       主斷路器
```

- 中文名寫「功能」而非廠牌型號（「主接觸器」而非「士林 S-P21」）；型號屬於零件清單（&EPB）的資訊。
- 字級 10 pt（與代號同級）；空間不足時 MAY 縮至 8 pt 或省略，但**代號永遠不可省略**——中文名是輔助，代號才是唯一識別。
- 同一功能的中文用詞全案一致（MUST）：開工前在代號慣例表中一併列出「代號－中文名」對照，之後全案沿用，不得同案混用「電磁接觸器／主接觸器／MC」等不同稱呼。
- 常用對照（預設值，可依案調整）：

| 代號 | 中文名 | 代號 | 中文名 |
|---|---|---|---|
| -Q | 主斷路器／隔離開關 | -S | 按鈕／選擇開關（急停：急停按鈕） |
| -F | 保護斷路器／熔絲 | -B | 感測器（依類型：近接／光電／壓力…） |
| -KM | 電磁接觸器 | -P／-H | 指示燈／蜂鳴器 |
| -KA | 輔助繼電器 | -X | 端子台 |
| -KS | 安全繼電器 | -U | 變頻器／伺服驅動器 |
| -T | 控制變壓器／電源供應器 | -A | PLC／控制器 |
| -M | 馬達 | -W | 電纜 |

### 1.2 類別字母表（MUST 使用下表，不得混用 JIS/JIC 縮寫如 MC、THRY、CR）

| 字母 | 元件類別 | 常見例 |
|---|---|---|
| Q | 動力電路開關／保護切換 | 斷路器（MCCB/MCB 動力）、電動機保護開關、隔離開關 |
| F | 保護裝置 | 熔絲、控制迴路斷路器、突波保護、積熱電驛（亦可用 -FR） |
| K | 繼電器、接觸器 | 電磁接觸器 KM 慣例可寫 -KM1，安全繼電器 -KA/-KS |
| T | 變壓器、變換器 | 控制變壓器、電源供應器（-T 或 -G 皆可，擇一並全案一致） |
| G | 電源供應裝置 | 發電機、DC 電源（若 T 已用於變壓器） |
| M | 電動機 | 馬達 |
| S | 操作／感測開關 | 按鈕、選擇開關、極限開關、急停（急停建議 -S0 或獨立序列） |
| B | 感測轉換器 | 接近開關、光電、編碼器、壓力傳送器 |
| P | 顯示裝置 | 指示燈、儀表、蜂鳴器（蜂鳴器亦可 -H） |
| H | 信號裝置 | 指示燈／聲響（與 P 擇一慣例，全案一致） |
| X | 端子、連接器 | 端子台 -X1，端子 -X1:5 |
| W | 電纜、導線 | 電纜 -W1 |
| A | 組合裝置 | PLC、驅動器、控制器模組 |
| U | 變流裝置 | 變頻器（VFD）、伺服驅動器（亦常見 -A，擇一全案一致） |
| E | 其他電氣 | 照明、加熱器 |

**一致性規則（MUST）**：同一專案內，同一種實體元件只能對應一種字母慣例。開始繪圖前先輸出「本案代號慣例表」，之後全案遵守。

## 2. 導線顏色與線號（IEC 60204-1）

### 2.1 顏色（依迴路類型指定 conductor 顏色屬性，MUST）

**圖面導線直接以實體導線規範色繪製**：QET conductor 的顏色屬性 MUST 設為下表對應值，使圖面線色＝現場配線色，配線師傅照圖即照色施工。禁止全案使用單一預設黑色繪製控制迴路。

| 迴路類型 | 顏色 | QET color 值 |
|---|---|---|
| AC／DC 動力主迴路 | 黑 | `#000000` |
| AC 控制迴路 | 紅 | `#FF0000` |
| DC 控制迴路（24VDC） | 深藍 | `#00008B` |
| 中性線 N | 淺藍 | `#ADD8E6` |
| 保護接地 PE | 綠/黃（QET 以綠表示＋標註 PE） | `#008000` |
| 主開關切斷後仍帶電之迴路 | 橘 ＋ 圖面警告標註 | `#FF8C00` |

- 淺藍（N）在白底列印時對比較弱，MAY 於頁面加註「N＝淺藍」圖例輔助；不得因此改用其他顏色繪製。
- 黑白影印件無法辨色，故線色是輔助資訊：線號與 PE 標註仍為識別的正式依據（此即 §2.2 線號為 MUST 的原因）。

- PE 導線與端子 **MUST** 標示 PE，不得挪作他用。
- 橘色迴路出現時，**MUST** 在該頁加註警告文字：「⚠ 本迴路於主開關 OFF 後仍帶電」。

### 2.2 線號（等電位編號）

- 每一等電位一個線號；線號在導線**兩端**皆須標示（QET conductor 的 formula/num 屬性）（MUST）。
- 跨頁同電位沿用同一線號（MUST）。
- 建議編號結構（SHOULD）：

| 區段 | 號段 | 例 |
|---|---|---|
| 三相動力 | L1/L2/L3、U/V/W ＋ 分支序號 | 1L1、2L1、U1 |
| AC 控制 | 100–199 | 101, 102… |
| DC 控制 +24V 側 | 200–299 | 201, 202… |
| DC 0V | 0V 或 300–399 | 0V1 |
| 類比信號 | 400–499 | 401 |
| PE | PE | PE |

- 經過元件（接點、線圈）後電位改變 → 線號**必須**改變；純分接點（同電位）→ 線號**不變**（MUST）。

## 3. 圖面結構（IEC 61082-1）

### 3.1 版面

- 控制迴路採階梯式：上橫線為電源正側（L / +24V），下橫線為負側（N / 0V），電流由上往下、迴路由左往右排列（MUST）。
- 動力迴路：三相垂直排列，由上（電源）至下（負載），相序 L1-L2-L3 由左至右（MUST）。
- 一頁一主題（SHOULD）：動力、控制、安全迴路、I/O 分頁繪製；單頁迴路行數超過版面 80% 即分頁。

### 3.2 交互參照（MUST）

- 格式：`頁次/欄列`，例 `3/B5`＝第 3 頁 B 列 5 欄。
- 接觸器／繼電器線圈下方放接點鏡像表（QET cross-reference 功能開啟），列出所有接點位置；接點旁標示線圈位置。
- 跨頁導線兩端成對標示：去向頁/座標＋線號，兩端線號一致。缺任一端即為錯誤。

### 3.3 符號（IEC 60617）

- 一律使用 IEC 60617 符號（QET 內建集合的 IEC 分支），**禁止**混用 JIS/ANSI 符號（MUST）。
- 接點以「未動作／斷電」狀態繪製（MUST）。
- 需要的符號不存在時：依 IEC 60617 風格新建 `.elmt`（端子落網格、線寬 normal），存入專案內嵌集合，並在回覆中列出新建清單（MUST）。

## 4. 安全迴路（IEC 60204-1 §9 / ISO 13850）

- 急停元件：紅色蘑菇頭 NC 接點符號，代號建議 -S0 系列；接點採直接開路動作（標註 ⊙ 符號者優先）（MUST）。
- 急停/安全功能 **MUST** 經安全繼電器或安全 PLC 迴路，不得僅以一般 PLC 輸入處理；繪製時安全繼電器輸出接點串入接觸器線圈迴路。
- 安全迴路頁 **MUST** 標註停止類別（Stop Category 0 或 1，依輸入需求；未指定時詢問使用者，不得自行假設）。
- 安全迴路導線顏色依其電源性質（AC 紅／DC 深藍），**不因**其為安全功能而改色。

## 5. 文件識別號與分類（ISO 7200 / IEC 61355）

### 5.1 編號結構（公司內規，MUST）

```
[公司碼]-[年度]-[專案/機台號]-[DCC]-[流水號]
例：HC-2026-001-&EFS-001
```

- 專案/機台號：**三碼阿拉伯數字**。
- 流水號：三碼，同專案同 DCC 內連續。
- 識別號發行後不可變更；內容修改走修訂索引升版。

### 5.2 DCC 代碼與交付組合

| DCC | 文件 | 何時需要 |
|---|---|---|
| &EFS | 電路圖 | 一律（核心文件） |
| &EPB | 零件清單 | 一律 |
| &ELD | 佈置圖 | 有控制盤即需 |
| &EMB / &EMA | 接線表／接線圖 | 二擇一（預設 &EMB 表格式） |
| &EFA | 概觀圖/單線圖 | 多電源、多盤互聯的大系統才需 |

未指定交付範圍時，預設產出 &EFS＋&ELD＋&EMB＋&EPB 四項的圖頁架構（SHOULD）。

## 6. 圖框與標題欄（ISO 7200 / ISO 5457）

### 6.1 頁面（MUST）

- A3 橫式；頁緣座標網格 8 欄（上下緣、數字、左→右）× 6 列（左右緣、字母、上→下）；格距約 200 px、邊條 40 px。

### 6.2 標題欄（置於圖框右下角，總寬 720 px；欄位小標籤 7 pt）

各橫排寬度合計 MUST ＝ 720 px。強制欄位（*）缺一即不合格。

| 排 | 欄位（寬 px／字體） |
|---|---|
| 1 | 技術參考 200／10pt ・ 繪製 200／10pt ・ 審核 160／10pt ・ 核准 160／10pt |
| 2 | 文件類別 200／10pt ・ 文件狀態 200／10pt ・ 發行日期* 160／10pt粗 ・ 修訂索引 160／10pt粗 |
| 3 | 文件識別號* 260／11pt粗 ・ 圖名 460／14pt粗（上）＋補充圖名 460／10pt（下） |
| 4 | 法定所有者* 460／14pt粗 ・ 頁次* 130／10pt粗 ・ 圖幅 130／10pt ・ 備註 260／10pt |

- 修訂記錄欄置於標題欄正上方，寬 720 px：版次 50／日期 100／修改內容 430／修改人 70／核准 70，列高 30 px，表頭 7 pt／內容 8 pt。
- QET 變數對應：圖名 `%{title}`、繪製 `%{author}`、發行日期 `%{date}`、頁次 `%{folio-id} / %{folio-total}`、修訂索引 `%{indexrev}`、識別號與所有者以專案自訂變數帶入（MUST 使用變數，不得在各頁硬編文字）。

### 6.3 修訂規則（MUST）

- 版次：發行前用數字（01, 02…）；首次正式發行＝A；後續 B、C…，**跳過 I、O、Q**。
- 每次升版三件事同步：標題欄修訂索引、修訂記錄欄新增一列（含位置參照如 3/B5）、發行日期更新（ISO 8601：YYYY-MM-DD）。
- 修改內容寫「改了什麼＋位置」，不寫原因。

## 8. 繪製工作流程（每次任務依序執行）

1. **釐清輸入**：確認機台名稱、專案號、電源規格（相數/電壓/頻率）、控制電壓、交付 DCC 範圍、停止類別。缺項→先詢問，**不得假設安全相關參數**。
2. **輸出代號慣例表**：本案使用的 81346 字母對應（§1.2 的擇一項目在此定案）。
3. **建立專案骨架**：圖框＋標題欄變數、頁面規劃（封面/目錄可選、動力、控制、安全、I/O、端子）。
4. **繪製**：依 §11.1 的頁序與繪製順序進行；放置元件、拉導線（含顏色與線號屬性）、加交互參照時遵守 §2–§4 與 §11.2–§11.4。
5. **自檢**：執行 §9 檢查清單，逐項回報 PASS/FAIL。
6. **交付**：`.qet` 檔＋自檢報告＋新建元件清單＋未決事項清單。

## 9. 自檢清單（輸出前 MUST 全數 PASS）

```
[ ] 所有元件端子座標為 10 的倍數
[ ] 所有元件具唯一代號，字母符合 §1.2，同類連續編號
[ ] 元件中文名稱與代號慣例表一致；代號無一省略
[ ] 所有導線具線號，且等電位一致、跨元件變號
[ ] 導線以規範色實際繪製（非全案預設黑），顏色與迴路類型對應（§2.1），橘色迴路附警告標註
[ ] PE 僅用於接地且有標示
[ ] 線圈具接點鏡像參照；跨頁導線兩端成對標示
[ ] 接點為斷電狀態繪製；符號皆為 IEC 60617
[ ] 急停迴路經安全繼電器；停止類別已標註
[ ] 標題欄四強制欄位齊備；識別號符合 §5.1 格式
[ ] 頁次 x/y 與實際頁數一致；修訂索引與修訂記錄欄對應
[ ] 標題欄使用變數而非硬編文字
[ ] 頁序符合 §11.1；每一迴路行僅含一個線圈／輸出
[ ] 每個功能群組具設計意圖註解；同型迴路版面一致
[ ] 備用端子與備用 I/O 已繪出並標「備用」；無斜線導線；超長導線已改為線號＋參照
[ ] §11.5 值班電工測試：每個輸出元件四個問題皆可由圖回答
```

## 10. 禁止事項（MUST NOT）

- 不得使用 JIS/JIC 代號（MC、THRY、CR…）或全黃控制線等台灣廠內慣例。
- 不得自創 DCC 代碼、自創端子編號。
- 不得在未確認風險評估結果的情況下自行決定停止類別或簡化安全迴路。
- 不得重用已作廢的圖號或元件代號。
- 不得為了排版美觀而讓端子脫離 10 px 網格。
- 自檢有 FAIL 項目時不得宣稱任務完成；須列出 FAIL 項與修正方案。

## 11. 版面與可維護性準則

> 本章目標：讓圖面對「三個月後的維護人員」而非只對繪圖當下的 AI 可讀。規範保證合法，本章保證好懂、好改。

### 11.1 頁面架構與繪製順序（MUST）

**頁面順序**（讀圖敘事：由電源入口往負載末端、由安全層往功能層）：

```
1. 封面／目錄（頁數 ≥5 時 MUST 有目錄頁）
2. 動力迴路（&EFS 動力部）
3. 控制電源（變壓器／電源供應器／控制迴路保護）
4. 安全迴路（急停鏈、安全繼電器、門鎖）
5. 控制迴路（依功能分頁，見 11.2）
6. PLC I/O 配置
7. 端子配置／接線資訊
```

**繪製順序**（MUST 依此序執行，不得先畫控制迴路）：

```
I/O 與元件清單 → 頁面規劃（目錄先定案） → 動力迴路 → 安全迴路 → 控制迴路 → 索引頁（I/O、端子、BOM）
```

- 先完成 I/O 與元件清單再開始繪圖：清單決定頁數、PLC 點數、端子規模；缺清單即開畫視為流程錯誤。
- 安全迴路 MUST 先於控制迴路繪製：安全繼電器輸出接點會串入後續所有接觸器線圈迴路，先定案其頁次座標，後續引用的交互參照才穩定。
- I/O 頁、端子頁、BOM 屬於「索引性內容」，MUST 於迴路內容全部定案後一次生成，改版時亦最後同步。

### 11.2 版面規則

| 規則 | 級別 | 機械判定條件 |
|---|---|---|
| 一行一因果：每一迴路行（rung）僅含一個線圈或輸出元件 | MUST | 單一垂直迴路內線圈/輸出計數 ≤1 |
| 一頁一主題：每頁僅一個功能群組（如「主軸啟停」「潤滑系統」） | SHOULD | 迴路行數超過版面 80% 即分頁 |
| 迴路行距 ≥40 px；功能群組間距 ≥80 px | MUST | 相鄰迴路 x 座標差檢核 |
| 每個功能群組上方置功能註解，寫「設計意圖」而非重述接線 | MUST | 例：「潤滑馬達：主軸運轉 10 s 後啟動」；禁止「K3 接 K4」這類無資訊註解 |
| 導線水平跨距 > 1/2 頁寬時，改為兩端線號＋交互參照，不拉實線 | SHOULD | 導線長度 >800 px 觸發 |
| 導線僅水平／垂直，禁止斜線；交叉數最小化 | MUST | 斜線＝FAIL |
| 佈局留白：單頁元件與導線佔用面積 ≤70% | SHOULD | 為修訂標記與後續增補預留空間 |

### 11.3 擴充預留（SHOULD，比例可依案指定）

- 端子台預留 ≥20% 備用端子，**畫進圖面**並標註「備用 Spare」——不畫等於不存在。
- PLC 每種 I/O 類型預留 ≥10% 備用點，圖面標「備用」。
- 線號依 §2.2 分段編號並於段內留空（例如每功能群組跳 10 號起編），插入新迴路時不需重編全案線號。
- 頁碼預留：功能區之間允許跳頁（…5、6、10、11…），供未來插頁；目錄如實列出。

### 11.4 一致性（MUST）

- **同型迴路版面完全複製**：同樣的馬達啟停迴路，不論出現在第幾頁，元件相對位置、順序、間距一致。首次繪製即建立版面範本，其後複製套用；禁止逐頁重新排版「最佳化」。
- 電源母線位置全案一致：控制頁上橫線＝L/+24V、下橫線＝N/0V，y 座標各頁相同。
- 元件標籤位置一致：代號在符號左上、中文名於其下、線號在導線上方——全案同一相對位置。

### 11.5 可讀性自測：「值班電工測試」（輸出前 MUST 執行）

情境：半夜停機，值班電工只有這份圖。對**每一個輸出元件**（接觸器、電磁閥、指示燈），AI 須能僅憑圖面回答以下四題；任一題答不出即為版面 FAIL，須補參照或註解：

```
Q1 它的線圈在哪一頁哪個座標？（目錄→頁→座標可達）
Q2 哪些條件（接點）驅動它？各接點的元件在哪？（鏡像表／參照完整）
Q3 它的電源迴路從哪一頁的哪個保護元件而來？（電源路徑可回溯）
Q4 要量測它，對應的端子台端子是哪幾個？（端子參照存在）
```

自測結果納入 §9 檢查清單第 15 項回報。
